# 架构设计图

以下示意图使用 **Mermaid**，可在 GitHub、GitLab、Typora、VS Code 等环境渲染。描述与当前仓库 **单体后端 + SPA 前端** 实现一致；微服务仅为可选演进方向。

## 1. 逻辑分层（C4 容器级简化）

```mermaid
flowchart TB
  subgraph client["客户端"]
    Browser["浏览器 / SPA\nReact + Vite + Ant Design"]
  end

  subgraph app["应用层"]
    API["FastAPI\n/api/v1"]
  end

  subgraph domain["领域服务层"]
    CS["ChatService\n问答/RAG/工具"]
    FS["FileService / KBService"]
    ES["Embedding / Vector / Search"]
    LS["LLMService"]
    EV["Evaluation / RAG Metrics"]
    ST["Steward / Computer / MCP"]
  end

  subgraph data["数据与外部依赖"]
    DB[("PostgreSQL / MySQL")]
    Redis[("Redis\n缓存/队列/限流")]
    VDB[("Zilliz / Qdrant\n向量库")]
    OSS[("MinIO / S3\n对象存储")]
    LLMExt["OpenAI 兼容 API\nEmbedding & LLM & Rerank"]
  end

  Browser -->|HTTPS JSON/SSE| API
  API --> CS & FS & ES & LS & EV & ST
  CS --> ES & LS & DB & Redis
  FS --> OSS & DB & ES
  ES --> VDB & LLMExt
  EV --> CS & ES & DB
  ST --> DB & Redis & LLMExt
```

## 2. 后端模块关系（简化）

```mermaid
flowchart LR
  subgraph api["app/api/v1"]
    R1["auth, files, knowledge_bases"]
    R2["chat, search, evaluation"]
    R3["steward, computer_steward, mcp, bash"]
    R4["billing, dashboard, audit, tasks"]
  end

  subgraph services["app/services"]
    S1["chat_service"]
    S2["advanced_rag_service"]
    S3["vector_store / embedding_service"]
    S4["recall_evaluation / rag_metrics"]
  end

  R1 & R2 & R3 & R4 --> services
  S1 --> S2 & S3
  S2 --> S3
```
