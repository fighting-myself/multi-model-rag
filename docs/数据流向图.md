# 数据流向图

描述主链路中的 **数据如何流动**；实现细节见 `backend/app/services/` 下各服务。

## 1. 文件入库与索引

```mermaid
sequenceDiagram
  participant U as 用户/前端
  participant API as FastAPI
  participant FS as FileService
  participant OSS as MinIO/S3
  participant DB as 关系库
  participant Q as 任务队列
  participant EMB as Embedding
  participant V as 向量库

  U->>API: 上传文件 (multipart)
  API->>FS: 保存元数据 + 存储
  FS->>OSS: 对象二进制
  FS->>DB: files 记录
  opt 异步解析
    FS->>Q: 解析/分块/向量化任务
    Q->>EMB: 分块文本/多模态
    EMB->>V: 向量 upsert
    Q->>DB: chunks / vector_id
  end
  API-->>U: 文件信息 + 状态
```

## 2. RAG 问答（流式）

```mermaid
flowchart LR
  subgraph input["输入"]
    Q["用户问题"]
    KB["知识库 ID 列表"]
    H["对话历史"]
  end

  subgraph retrieve["检索"]
    E["Query Embedding"]
    V["向量检索"]
    B["BM25 / 全文"]
    F["RRF 融合 + Rerank"]
  end

  subgraph gen["生成"]
    C["上下文拼接"]
    L["LLM 流式输出"]
  end

  subgraph out["输出"]
    S["SSE/流式文本"]
    SRC["命中片段/溯源"]
  end

  Q & KB & H --> E
  E --> V & B --> F --> C --> L --> S
  F --> SRC
```

可选分支（由配置与请求参数决定）：

- **Advanced RAG**：多查询改写后再检索。
- **联网检索**：工具/搜索服务结果注入上下文。
- **工具/MCP**：LLM 或编排层调用外部工具后再回答。

## 3. 非流式补全

与流式相同，仅在 `ChatService` 出口处 **一次性返回** 完整文本，无 SSE 分块。
