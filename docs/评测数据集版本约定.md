# 评测数据集版本约定（改造 C-4）

> 与 `recall_evaluation_service`、`benchmark_datasets` 表及前端评测页配合使用，便于回归对比。

## 约定

1. **逻辑版本**：在业务上为每组 benchmark 问题集维护一个 **版本号**（如 `v1`、`2026-04-01`），变更题目或标准答案时递增版本。
2. **库表**：`benchmark_datasets`（若已使用）可在 `name` 或扩展列中体现版本后缀，或在应用层用「数据集名 + 版本」唯一标识一次评测快照。
3. **文档记录**：重大变更（题目增删、chunk 标注变更）在 PR 或本仓库 `docs/` 中记一行摘要，便于面试与审计时说明「指标可复现条件」。
