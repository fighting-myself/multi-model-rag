# RAG 与依赖降级路径（改造 D-3）

> 便于排障与面试说明：**失败时系统如何表现**，日志关键字与行为对齐。

| 场景 | 行为 | 日志/可观测 |
|------|------|-------------|
| Redis 不可用 | `cache_service` get/set 失败 → 视为未命中；限流可能不生效 | `缓存 Redis 连接失败` / `限流将不生效` |
| Advanced RAG 异常 | 回退到普通混合检索或提示文案 | `Advanced RAG 检索失败` |
| Rerank API 失败 | 使用 RRF 顺序截断 Top-K | `Rerank 失败` / `召回评测 Rerank 失败` |
| 向量检索无结果（单库） | 取库内前若干 chunk 兜底或空上下文提示 | 进度回调「向量检索无可用向量 id」 |
| DashScope Embedding 超时 | 可配置次数内重试；仍失败则抛错 | `DashScope embedding 网络超时`；指标 `embedding_transport_retries_total`（见 `/api/v1/ops/snapshot`） |
| LLM HTTP 超时 | 由 OpenAI SDK 超时与重试策略失败；上层捕获 | 配置见 `LLM_HTTP_*` |
